Reglas del proyecto

• Nunca crear componentes mayores a 300 líneas.
• Cada módulo debe tener su carpeta.
• Nunca duplicar lógica.
• Los formularios usan React Hook Form + Zod.
• Todas las consultas a Firestore pasan por services/.
• Nunca consultar Firestore directamente desde los componentes.
• Todo componente debe ser responsive.
• Tailwind únicamente.
• No usar CSS tradicional salvo estilos globales.
• No instalar librerías sin autorización.
• Mantener consistencia visual en todo el sistema.
